# Bobathon — 실무자 120분 랩 운영 설계

IBM Bob 고객 초청 해커톤의 **운영 설계 문서 + 랩 자산**입니다.
개발 실무자 30~50명, 단일 세션, **14:30–16:30 (120분)**.

- **운영 설계 문서**: [`index.html`](index.html) — 퍼실리테이터가 위에서 아래로 읽는 단일 페이지
- **랩 자산**: [`lab-assets/`](lab-assets/) — 스타터 레포에 그대로 얹는 파일들

---

## 이전 설계와 무엇이 다른가

기존 `bobathon-prep`은 **데모 3개(20 · 10 · 20)** 로 앞 60분을 채웠고, 그 문서 스스로가 최대 리스크를 이렇게 적어 두었습니다.

> "참가자가 60분 동안 가만히 앉아만 있다가, 갑자기 20분 만에 아이디어를 내야 한다."

이 설계는 그 60분을 **직접 손으로 하는 랩 3개**로 바꿉니다. 보는 사람은 아이디어가 안 나오지만, **한 번 해 본 사람은 자기 업무가 저절로 떠오릅니다.**

시간 예산이 120분 그대로이므로, **아이데이션을 30분 워크숍이 아니라 10분 스코프 게이트로** 줄이고 그 시간을 빌드 스프린트에 넘깁니다.

| | 이전 | 이번 |
|---|---|---|
| 앞 60분 | 데모 3개 (보기) | **랩 3개 (직접)** |
| 아이데이션 | 20분 워크숍 | **10분 스코프 게이트** |
| 실행 | 30분 | 30분 (동일) |
| 15:35 시점의 참가자 | 메모 3줄 | **작동하는 `.bob/` 디렉터리** |

## 표어와 심사 기준

### "채팅이 아니라 자산을 내놓는다"

이 방의 실무자들은 "AI가 코드를 짜준다"를 **이미 압니다.** 그건 팔 것이 못 됩니다.
Bob의 차별점은 **레포 안에 살면서, 커밋되고 팀에 공유되는 자산을 만든다**는 것입니다 — `AGENTS.md`, Skill, 커스텀 모드, MCP 서버.

그래서 **문서·리포트 생성도 정당한 Bobathon 과제입니다.** 밸류가 없는 것은 문서 작업이 아니라 **일회성 채팅 답변**입니다. 레포를 읽어 매주 구조화된 리포트를 뱉는 Skill은, 생성된 API 엔드포인트와 **동등한 등급의 산출물**입니다. (IBM 공식 튜토리얼 `ide/tutorials/audit-code`가 문자 그대로 이것입니다: audit Skill → `findings.md` → SARIF/OSCAL 리포트.)

### 심사 기준은 단 하나

> **"이걸 월요일에 우리 팀 레포에 커밋하겠습니까?"**

이 한 문장이 코드 자산과 문서 자산을 **같은 링 위에 올립니다.**

## 스타터 레포 — **빌드 완료 · 검증 완료**

랩 3개는 **단일 레포**에서 누적 진행합니다. `IBM/galaxium-travels` (`demo/java-premium-legacy`, Apache-2.0)를 포크해 재구성했습니다.

| 산출물 | 무엇 |
|---|---|
| `../bobathon-starter/` | 작업본 (퍼실리테이터용 — `facilitator/` 포함) |
| `../bobathon-dist/bobathon-starter.zip` | **참가자 배포본** · 30MB · 오프라인 번들 포함 · 커밋 1개 |
| `../bobathon-dist/bobathon-facilitator.zip` | **정답지 + 스포일러 + 런북.** 참가자에게 주지 마세요 |

레포 안의 랩 자산: `lab/LAB{1,2,3}.md` (치트시트) · `CONVENTIONS.md` · `.bob/*.example` · `.bob/skills/legacy-safety-net/` · `scripts/preflight.sh` · `BOBATHON.md`

### 실측으로 검증한 것

깨끗한 임시 디렉터리에 **zip을 풀어서** 전부 돌렸습니다.

| 검증 | 결과 |
|---|---|
| LAB 1 — `pytest` (오프라인, 동봉 휠) | **72개 통과** |
| LAB 2 — MCP 서버 stdio 스모크 | **`list_tickets` 동작** (빌드 0 · 네트워크 0) |
| LAB 2 ② — `create_ticket` 정답 적용 | **빌드 → 호출 → `tickets.json` 실제 증가** 확인 후 되돌림 |
| LAB 3 — `./mvnw -o test` (오프라인) | **JDK 8 · 21 · 23 전부 통과** → **JDK 고정 불필요** |
| LAB 3 출발점 | 테스트 0개 → `No tests were executed!` **BUILD FAILURE** (정상) |
| 스포일러 격리 | 참가자 zip·git 히스토리에 정답지 **0건** |
| `mvnw` 실행권한 | zip 통과 후에도 살아있음 |
| 프리플라이트 | **12개 항목 전부 초록** |

### 가장 중요한 함정

> **`./mvnw -o`는 오프라인이 아닙니다.** `-o`는 Maven에게 전달될 뿐,
> **래퍼 스크립트는 그 전에 Maven 배포판 9MB를 받으러 갑니다.**
> → **D-5에 참가자 각자가 `scripts/preflight.sh`를 자기 사내망에서 한 번 돌려야** 합니다.

> **랩을 죽이는 1순위:** Bob이 `@SpringBootTest` / WireMock에 손대면 새 의존성 → **오프라인 즉사.**
> SKILL.md와 치트시트에 "Mockito 단위 테스트만, pom 건드리지 마"를 못박아 뒀습니다.
